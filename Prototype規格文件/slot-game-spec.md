> **目標**：製作可在手機瀏覽器上遊玩的 Demo，供客戶 / 投資人展示。

## 3. 遊戲盤面

| 項目 | 數值 |
|------|------|
| 列數（Reels） | 5 |
| 行數（Rows） | 3 |
| Payline 數量 | 15 條固定 Payline |
| 中獎方向 | **右→左**（由第 5 列向第 1 列連線） |

### 15 條 Payline 定義（格子編號：列 C1~C5，行 R1~R3）

以下為標準 5×3 Slot 常見固定線路，中獎方向均為 右→左：

| 線號 | 路徑（由右至左） |
|------|----------------|
| 1  | R2-C5, R2-C4, R2-C3, R2-C2, R2-C1 （中橫線） |
| 2  | R1-C5, R1-C4, R1-C3, R1-C2, R1-C1 （上橫線） |
| 3  | R3-C5, R3-C4, R3-C3, R3-C2, R3-C1 （下橫線） |
| 4  | R1-C5, R2-C4, R3-C3, R2-C2, R1-C1 （V 形） |
| 5  | R3-C5, R2-C4, R1-C3, R2-C2, R3-C1 （倒 V 形） |
| 6  | R1-C5, R1-C4, R2-C3, R3-C2, R3-C1 （向下階梯） |
| 7  | R3-C5, R3-C4, R2-C3, R1-C2, R1-C1 （向上階梯） |
| 8  | R2-C5, R1-C4, R1-C3, R2-C2, R3-C1 |
| 9  | R2-C5, R3-C4, R3-C3, R2-C2, R1-C1 |
| 10 | R1-C5, R2-C4, R2-C3, R2-C2, R1-C1 |
| 11 | R3-C5, R2-C4, R2-C3, R2-C2, R3-C1 |
| 12 | R1-C5, R1-C4, R2-C3, R2-C2, R3-C1 |
| 13 | R3-C5, R3-C4, R2-C3, R2-C2, R1-C1 |
| 14 | R2-C5, R1-C4, R2-C3, R3-C2, R2-C1 |
| 15 | R2-C5, R3-C4, R2-C3, R1-C2, R2-C1 |

> 中獎判斷：在該線從第 5 列開始，連續相同符號（可含 WILD 替代）最少 3 個以上。

## 4. 符號系統

### 4.1 一般符號（共 5 種）

| 類型 | 主題設定（像素怪物） | 倍率（3個 / 4個 / 5個） |
|------|-------------------|-----------------------|
| 低分符號 A | 低階小怪（例：哥布林） | 5× / 10× / 20× |
| 低分符號 B | 低階小怪（例：蝙蝠） | 5× / 10× / 20× |
| 低分符號 C | 低階小怪（例：骷髏兵） | 8× / 15× / 30× |
| 高分符號 D | 高階精英怪（例：暗黑騎士） | 15× / 30× / 60× |
| 高分符號 E | 高階精英怪（例：巫妖） | 20× / 40× / 80× |

> 倍率基準為「單線投注額」，可依平衡需求調整。

### 4.2 WILD 符號（共 2 種）

| 類型 | 外觀（像素） | 替代範圍 | 有效次數 |
|------|------------|---------|---------|
| 小 WILD | 魔法陣（小） | 可替代：低分符號 A/B/C + 其他 WILD | 2 次後失效 |
| 大 WILD | 魔法陣（大） | 可替代：所有符號（包含高分 D/E） | 2 次後失效 |

**失效規則**：WILD 每次被計入有效連線算 1 次，累計 2 次後，該格符號視覺標記為「灰暗 / 破碎」狀態，不參與後續連線判斷。

## 5. 核心遊戲機制：消除式（Cascade / Tumble）

### 5.1 每局流程

```
玩家按下 SPIN
    │
    ▼
系統預先計算本局所有結果（含連鎖消除後各步的盤面）
    │
    ▼
符號「由下往上」逐格出現在盤面（像素怪物從地底冒出動畫）
    │
    ▼
判斷所有 15 條 Payline 是否中獎（右→左）
    ├── 無中獎 → 本局結束
    └── 有中獎
           │
           ▼
       顯示中獎連線特效（右→左光束）
       魔法師施法動畫 / BOSS 受擊動畫
           │
           ▼
       加入獎金
           │
           ▼
       中獎符號消失（以概率觸發 WILD 掉落，見 5.2）
           │
           ▼
       新符號在空格「由下往上」補入
           │
           ▼
       再次判斷中獎（連鎖）
           │
           └── 直到無新中獎 → 本局結束
```

### 5.2 WILD 掉落機制

| 觸發條件 | 掉落 WILD 類型 | 建議觸發機率 |
|---------|--------------|-------------|
| 低分符號（A/B/C）被消除後 | 小 WILD | 約 15%（可調） |
| 高分符號（D/E）被消除後 | 大 WILD | 約 10%（可調） |

- WILD 掉落位置：與被消除符號相同格位
- WILD 出現動畫與一般符號相同（由下往上冒出）
- WILD 掉落後立即參與下一輪中獎判斷
- WILD 失效後（使用 2 次），下一輪消除中才被移除，**不再觸發新 WILD**

### 5.3 符號出現動畫規格

- **無假輪轉**：不做傳統捲軸滾動動畫
- 符號從格子底部往上「浮現」，配合像素逐格顯示效果
- 各格出現時間可略有交錯（建議：以列為單位，從第 1 列到第 5 列依序略有延遲）
- 動畫時長建議：單格出現約 200~300ms

## 6. 投注系統

| 項目 | 設定 |
|------|------|
| 籌碼面額 | 1 / 5 / 10 / 20 |
| 總注額計算 | 所選面額 × 15（固定 15 條線） |
| 起始餘額 | 10,000（建議值，可調） |
| 餘額顯示 | 即時更新，保留整數 |
| SPIN 消耗 | 每次 SPIN 前扣除總注額 |
| 中獎加分 | 各線獎金加總，加入餘額 |
| 餘額不足 | 按鈕 disable，提示「餘額不足」 |

## 7. 視覺 / 動畫規格

### 7.1 符號動畫

| 事件 | 動畫效果 |
|------|---------|
| 符號出現 | 由下往上冒出（像素逐行顯示） |
| 一般中獎 | 中獎符號放大 → 縮小（scale 1.0 → 1.3 → 1.0） |
| WILD 中獎 | 更強烈的放大 + 發光效果（scale 1.0 → 1.5 → 1.0 + glow） |
| 符號消除 | 淡出 / 像素碎裂消失 |
| WILD 失效 | 符號變灰暗（desaturate 濾鏡） |

### 7.2 中獎連線特效

- 連線高亮從**右側（第 5 列）往左側（第 1 列）**移動
- 以光束 / 魔法軌跡方式呈現（象徵魔法師攻擊 BOSS 防護罩）
- 一般連線：細光束，淡藍色或白色
- WILD 連線：粗光束，金色或紫色，帶粒子效果

### 7.3 角色動畫（像素動畫，至少 2~4 幀循環）

#### 魔法師（右側）

| 狀態 | 動畫內容 |
|------|---------|
| 待機（Idle） | 輕微呼吸起伏、斗篷飄動 |
| 施法（Cast） | 舉起魔杖、發射魔法光束向左 |

#### BOSS（左側）

| 狀態 | 動畫內容 |
|------|---------|
| 待機（Idle） | 緩慢移動、眼睛發光等 |
| 防護罩受擊 | 防護罩閃光 / 震動效果（疊加在 BOSS 前方） |

#### 動畫觸發對照

| 遊戲事件 | 魔法師 | BOSS |
|---------|-------|------|
| SPIN 中（等待符號出現） | Idle | Idle |
| 任何中獎 | 施法動畫 | Idle + 防護罩受擊 |
| WILD 連線中獎 | 施法（強版） | 防護罩受擊（更強烈閃光） |

## 8. 音效規格

| 音效 | 觸發時機 | 備註 |
|------|---------|------|
| 按鈕點擊音 | 按下任何按鈕 | 短促清脆 |
| 符號出現音 | 符號冒出時 | 可整批觸發一次 |
| 中獎音 | 確認中獎後 | 歡快魔法音效 |
| WILD 中獎音 | WILD 參與連線 | 更強烈、更有層次感 |
| 符號消除音 | 符號消除動畫 | 輕微爆破 / 消散音 |

> **BGM**：若網頁資源允許，加入循環背景音樂（中古奇幻風格）。BGM 為選配，不影響核心功能。

所有音效建議使用 **Web Audio API** 程式生成（避免外部音效檔依賴），或使用極輕量的 Base64 內嵌音效。

## 11. 美術資源方式（推薦）

由於是 Prototype，建議以下方案之一：

**方案 A（推薦）：程式生成像素圖**
- 使用 Canvas 繪製像素風格的符號（簡單幾何 + 顏色區分）
- BOSS / 魔法師以像素陣列（pixel array）定義逐格動畫幀
- 無需外部圖片檔，HTML 單檔即可運作

**方案 B：Base64 內嵌 PNG**
- 用像素繪圖工具製作小尺寸圖（例如 32×32 px），轉 Base64 嵌入 HTML
- 視覺更精緻，但需額外製作圖稿

> **建議開發時採用方案 A 快速出 Prototype，視覺驗收後再換方案 B 圖稿。**
